Gulp is a

kickass

task runner...

With it, you minify your .css file this way: 

            
              gulp.task('minify-css', function () {
  return gulp.src('./assets/css/screen.css')
    .pipe(minifyCss())
    .pipe(gulp.dest('./static/css/'));
});

or if you need to minify multiple files, just use an []: 

            
gulp.task('minify-css', function () {
  return gulp.src(['./assets/css/screen.css', './assets/css/mobile.css'])
    .pipe(minifyCss())
    .pipe(gulp.dest('./static/css/'));
});

Sexy and simple, huh?

But you are already familiar with this simplicity.

The point for today is deeper.

The point for today will give you an idea about decoupling tasks on Gulp, using **Browserify**, **SASS**, **Concat**, **Minfication** and **React** as example.

Promising, huh? Let's get started, then...

On Gulp, some people do the following: 

            
var gulp        = require ('gulp')
    , concatCss = require ('gulp-concat-css')
    , minifyCss = require ('gulp-minify-css');

// Your other tasks...

gulp.task('css', function () {
  return gulp.src(
      ['./assets/css/screen.css', './assets/css/mobile.css']
    ).pipe(concatCss('package.css'))
    .pipe(minifyCss())
    .pipe(gulp.dest('./static/css/'));
});

// Your other tasks (now maybe your 'default' task)

Now, let me ask you something:

You saw something wrong?

Technically, nothing's wrong.

But conceptually, I think so.

See:

              
var cssFiles = [
  './assets/css/screen.css', './assets/css/mobile.css'
];

gulp.task('concat-css', function () {
  return gulp.src(cssFiles)
    .pipe(concatCss('package.css'))
    .pipe(gulp.dest('./static/css/'));
});

gulp.task('minify-css', function () {
  return gulp.src('./static/css/package.css')
    .pipe(minifyCss())
    .pipe(gulp.dest('./static/css/'));
});

gulp.task('css', ['concat-css', 'minify-css']);

Could you feel the decoupling?

Hell yeah!

The decoupling transformed the tasks into three different natures which you can invoke independently based on your need, being the unique loss the weight of **gulpfile.js**.

By example:

That **gulpfile.js**...

without decoupling:

427 bytes

with decoupling:

621 bytes

Of course, the difference get bigger when your gulpfile gain tasks. But take it easy because Gulp is server-side and overall the file weight increase is insignificant.

What does fundamental mean?

Until now, the applicability of tasks decoupling on Gulp are being a little abstract, ain't it?

This is because decoupling tasks get more useful when you have a real scenario of appliance. Let's bootstrap with SASS.

Designing SASS as a Gulp's task: 

            
gulp.task('sass', function () {
  return gulp.src('./assets/scss/*')
    .pipe(sass())
    .pipe(gulp.desc('./assets/css/'));
});
